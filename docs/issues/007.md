# 007 — Chat 相關 TypeScript 編譯錯誤（4 項）

## 問題描述

啟動 `npm start` 出現以下錯誤：

```
Module not found: Error: Can't resolve 'socket.io-client'

TS2322: Property 'style' does not exist on type 'IntrinsicAttributes & ButtonProps'

TS2322: Property 'disabled' does not exist on type 'IntrinsicAttributes & ButtonProps'

TS2322: Type '(e: ChangeEvent<HTMLInputElement>) => void' is not assignable to type
        '(event: ChangeEvent<HTMLInputElement> | ChangeEvent<HTMLTextAreaElement>) => void'
```

---

## 根本原因

### 錯誤一：`socket.io-client` 未安裝

`package.json` 中已列出 `"socket.io-client": "^4.8.1"` 但未執行 `npm install`。

### 錯誤二、三：`ButtonProps` 缺少 `style` 與 `disabled`

`src/common/types.ts` 的 `ButtonProps` 未定義這兩個屬性，`Chat/index.tsx` 使用時報錯：

```tsx
<Button onClick={onLeave} style={{ padding: "2px 10px", fontSize: 12 }}>   // style 報錯
<Button disabled={!connected || !inputValue.trim()} onClick={handleSend}>   // disabled 報錯
```

### 錯誤四：`InputProps.onChange` 聯集型別過寬

`InputProps.onChange` 被定義為 `ChangeEvent<HTMLInputElement> | ChangeEvent<HTMLTextAreaElement>`，
但 `Input` 元件只 render `<input>`，不涉及 textarea。
TypeScript 的逆變性（contravariance）規定：只處理 `HTMLInputElement` 的函式不能指派給需同時處理兩種元素的型別。

---

## 相關檔案

| 檔案 | 說明 |
|------|------|
| `package.json` | 缺少 `npm install` |
| `src/common/types.ts` | `ButtonProps`、`InputProps` 型別定義 |
| `src/common/Button/index.tsx` | `Button` 元件未傳入 `style`、`disabled` |
| `src/common/Input/index.tsx` | `Input` 元件未傳入 `onKeyDown` |
| `src/components/Chat/index.tsx` | 使用 `style`、`disabled`、`onKeyDown` |
| `src/components/ChatEnter/index.tsx` | `onChange` 型別不符 |

---

## 修復內容

1. `npm install` — 補裝 `socket.io-client`
2. `ButtonProps` 新增 `disabled?: boolean` 與 `style?: React.CSSProperties`，並在 `Button/index.tsx` 傳入 `StyledButton`
3. `InputProps` 新增 `onKeyDown?: React.KeyboardEventHandler<HTMLInputElement>`，並在 `Input/index.tsx` 傳入 `StyledInput`
4. `InputProps.onChange` 改為 `(event: React.ChangeEvent<HTMLInputElement>) => void`，反映 `Input` 元件只渲染 `<input>` 的實際情況；傳入較寬型別的 handler（如 `useForm.handleChange`）仍符合逆變性規則，不需修改呼叫端
