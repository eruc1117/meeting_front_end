# 006 — 前端錯誤訊息三處問題

## 問題描述

1. 後端回傳 `E013_WEAK_PASSWORD` 時，前端 `ERROR_FIELD_MAP` 沒有對應項目，錯誤顯示在通用訊息欄而非密碼欄位
2. 前端密碼驗證只要求 6 個字元，後端驗證規則是 8 個字元 + 大小寫 + 數字，導致前端驗證通過但後端拒絕
3. `PUT /api/auth/updatePassword` 缺少 `Authorization` header，路由掛有 `authMiddleware` 必定回傳 401

---

## 根本原因

### 問題一：`E013_WEAK_PASSWORD` 未映射

`Login/index.tsx` 的 `ERROR_FIELD_MAP` 僅列出舊有錯誤碼，後端後來新增 `E013_WEAK_PASSWORD` 時前端未同步更新：

```ts
// 修改前
const ERROR_FIELD_MAP = {
  E001_USER_EXISTS: "account",
  E002_INVALID_EMAIL: "email",
  E009_PASSWORD_NOT_SAME: "passwordChk",
  E008_ACCOUNT_NOT_EXIST: "account",
  E003_INVALID_CREDENTIALS: "password",
  // E013_WEAK_PASSWORD 漏掉
};
```

### 問題二：密碼強度驗證不一致

```ts
// 前端（修改前）
if (!formData.password || formData.password.length < 6) {
  newErrors.password = "密碼至少需要 6 個字元";
}
// 後端（AuthService.js）
if (password.length < 8 || !/[A-Z]/.test(password) || ...) { ... }
```

### 問題三：缺少 Authorization header

```ts
// 修改前（userForm/index.tsx）
const res = await fetch(`${BASE_URL}/api/auth/updatePassword`, {
  method: "PUT",
  headers: { "Content-Type": "application/json" },  // 缺少 Authorization
  body: JSON.stringify({ account: values.account, ... }),  // 多送 account（後端不使用）
});
```

## 相關檔案

| 檔案 | 說明 |
|------|------|
| `src/components/Login/index.tsx` | `ERROR_FIELD_MAP`、密碼前端驗證 |
| `src/components/userForm/index.tsx` | `updatePassword` API 呼叫 |
| `backend/services/AuthService.js` | `validatePassword` 實作（8 chars + 大小寫 + 數字） |
| `backend/routes/auth/auth.js` | `PUT /updatePassword` 掛 `authMiddleware` |

---

## 修復內容

1. `ERROR_FIELD_MAP` 新增 `E013_WEAK_PASSWORD: "password"`
2. 登入時只驗空白；**註冊**時加入完整複雜度驗證（8 chars + 大小寫 + 數字）
3. `updatePassword` 加入 `Authorization: Bearer ${token}` header，移除多餘的 `account` 欄位
