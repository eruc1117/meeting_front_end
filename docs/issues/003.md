# 003 — 點選 Schedule 跳轉至登入頁（AuthContext Race Condition）

## 問題描述

已登入的使用者點選 Header 的「Schedule」連結後，被導向 `/login` 而非 `/schedule`。

---

## 根本原因

**AuthContext 初始化 Race Condition**

`src/contexts/AuthContext.js` 中：

```js
const [isLoggedIn, setIsLoggedIn] = useState(false);  // 初始值 false

useEffect(() => {
  const token = localStorage.getItem("token");
  // ... 讀取 localStorage 並 setIsLoggedIn(true)
}, []);
```

`useEffect` 在 **第一次 render 之後** 才執行。
`src/router/index.tsx` 的 `PrivateRoute` 在第一次 render 時讀取 `isLoggedIn`，此時 `useEffect` 尚未完成，`isLoggedIn` 仍為 `false`，所以直接 `<Redirect to="/login" />`。

---

## 問題流程

```
使用者進入 /schedule
  → PrivateRoute render（第 1 次）
  → isLoggedIn = false（useEffect 尚未執行）
  → Redirect to /login  ← 錯誤發生

  → useEffect 執行，讀到 localStorage token
  → setIsLoggedIn(true)  ← 但已經跳轉了
```

---

## 相關檔案

| 檔案 | 行數 | 說明 |
|------|------|------|
| `src/contexts/AuthContext.js` | 8, 11–25 | `isLoggedIn` 初始值 false，useEffect 才還原 session |
| `src/router/index.tsx` | 11–22 | PrivateRoute：第一次 render 就判斷 isLoggedIn |

---

## 次要問題

`Login/index.tsx` 第 104 行：登入成功後執行 `history.push("/home")`，但路由中沒有 `/home` 這個頁面（應為 `/` 或 `/schedule`）。

---

## 修復方向

在 `AuthContext` 加入 `isInitialized` 狀態：

```js
const [isInitialized, setIsInitialized] = useState(false);

useEffect(() => {
  // ... 讀 localStorage ...
  setIsInitialized(true);  // 無論有無 token 都設為 true
}, []);
```

`PrivateRoute` 改為：

```jsx
if (!isInitialized) return null;  // 或 loading spinner
return isLoggedIn ? <Component {...props} /> : <Redirect to="/login" />;
```
