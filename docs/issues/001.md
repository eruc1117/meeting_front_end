# 001 — 行事曆建立活動失敗

**日期**: 2026-02-28
**狀態**: ✅ 已解決
**問答紀錄**: [問答紀錄.md](./問答紀錄.md)

---

## 問題描述

點擊「新增」按鈕建立行事曆事件時，console 出現：

```
index.tsx:71 Failed to create schedule: Error: 活動建立失敗，資料格式錯誤
  at createSchedule (SechduleContext.js:69:1)
  at async handleCreateSchedule (index.tsx:68:1)
```

---

## 根本原因（複數）

調查後發現此次問題實為多個獨立缺陷同時存在：

| # | 問題 | 位置 |
|---|------|------|
| A | 按鈕缺少 `type="button"`，每次點擊同時觸發 `onClick` 與 `onSubmit`，重複呼叫 API | `scheduleForm/index.tsx` |
| B | `UserContext.js` 為舊版重複 Context，所有路由停留在舊版（包含 GET 方法查詢） | `contexts/UserContext.js` |
| C | `scheduleDisplay` 表格欄位 key 與 API 回傳欄位名稱完全不符，顯示空白 | `scheduleDisplay/index.tsx` |
| D | `getSchedules` 的 `user_id` 未轉為 Number | `SechduleContext.js` |
| E | 新增用的 `createSchedule` 的 `user_id` 未轉為 Number | `scheduleForm/index.tsx` |
| F | API 路由更新：建立/更新/刪除端點路徑已變更 | `SechduleContext.js`, `UserContext.js` |

> **關於原始錯誤訊息**：後端 AI 確認 HTTP 401 + E011 的組合**不可能同時出現**。401 表示 JWT 在 middleware 層被攔截，不會執行到 Service 層回傳 E011。原始報錯可能是兩次不同請求的錯誤被混合解讀，或由重複 API 呼叫造成。

---

## 修改清單

### 1. `src/components/scheduleForm/index.tsx`
- `user_id: Number(user.id)`（建立活動）
- 所有 `onClick` 按鈕加上 `type="button"`，防止觸發 form `onSubmit`
- 「查詢空閒」加上 `type="submit"`（明確標記）

### 2. `src/contexts/SechduleContext.js`
- `getSchedules`：`user_id: Number(userId)`
- 路由更新：
  - `POST /api/schedules` → `POST /api/schedules/create`
  - `PUT /api/schedules/:id` → `PUT /api/schedules/update/:id`
  - `DELETE /api/schedules/:id` → `DELETE /api/schedules/delete/:id`

### 3. `src/contexts/UserContext.js`（舊版 Context，仍保留以維持相容）
- 路由全面同步至新版（同上）
- `getSchedules`：`GET /api/schedules` → `POST /api/schedules/query`，補上 `user_id` body 及完整 mapSchedule

### 4. `src/components/scheduleDisplay/index.tsx`
- 表格欄位 key 修正：
  - `date` → `startTime`
  - `Time` → `endTime`
  - `place` → `location`
  - `attend` → `participants`

---

## 後端 AI 確認事項

| 項目 | 結果 |
|------|------|
| `user_id` 字串型別 | 後端可接受（pg driver 自動 coerce），但前端仍統一轉 Number |
| datetime 格式 `YYYY-MM-DDTHH:MM:ss` | 後端接受，不需時區後綴 |
| 選填欄位為 `undefined` | 後端以 `?? null` 處理，存入 NULL，不影響 |
| 真正錯誤原因 | JWT Token 未帶入或已過期（HTTP 401）|
