# 005 — 點擊搜尋結果後，結果數量發生變化

## 問題描述

使用者點擊搜尋結果後，結果列表摺疊，但重新展開時顯示的筆數與原本搜尋時不同。

---

## 根本原因

**`searchResults` 是 live 計算，`tableData` 被覆蓋導致數量改變。**

流程：

```
使用者點擊結果
  → handleResultClick
  → setJumpDate(...)           ← Calendar 收到新的 jumpToDate
  → setResultsCollapsed(true)  ← 結果列表摺疊

  → Calendar useEffect([jumpToDate]) 觸發
  → setViewDate(新月份)         ← 行事曆跳轉

  → Calendar useEffect([year, month, user.id]) 觸發
  → getSchedules(userId, 新月份 startISO, 新月份 endISO)
  → tableData 被覆蓋為「新月份的資料」  ← 問題根源

  → scheduleForm 中 searchResults 重新計算
  → 結果數量變成新月份篩選後的筆數（非原搜尋結果）
```

---

## 相關程式碼

| 檔案 | 行為 |
|------|------|
| `scheduleForm/index.tsx` | `searchResults` 直接從 `tableData` live 計算 |
| `Calendar/index.tsx` | `useEffect([year, month, user.id])` → 跳轉月份時呼叫 `getSchedules` 覆蓋 `tableData` |
| `ScheduleContext.js` | `getSchedules` 成功後 `setTableData(...)` 直接替換整個列表 |

---

## 修復方向

在 `handleSearch` 執行時，將搜尋結果**快照**儲存至獨立 state（`snapshotResults`），不再 live 依賴 `tableData`：

```tsx
// 修改前（live 計算）
const searchResults = showResults
  ? (tableData as any[]).filter(...)
  : [];

// 修改後（快照）
const [snapshotResults, setSnapshotResults] = useState<any[]>([]);

// 在 handleSearch 中，等 getSchedules 完成後儲存快照
const handleSearch = async () => {
  const data = await getSchedules(user?.id, startISO, endISO);
  const filtered = (data ?? []).filter(...);
  setSnapshotResults(filtered);
  setShowResults(true);
  setResultsCollapsed(false);
};
```

**注意：** `getSchedules` 目前回傳 API response，需確認其回傳的 data 格式可直接使用，或從更新後的 `tableData` 取值（需考慮非同步時序）。
