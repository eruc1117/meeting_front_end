# 002 — Calendar `setSubMenu` 殘留參照 TS2304 錯誤

## 日期
2026-03-03

## 狀態
✅ 已解決

---

## 問題描述

執行 `npm start` 時，TypeScript 編譯失敗：

```
ERROR in src/components/Calendar/index.tsx:122:47
TS2304: Cannot find name 'setSubMenu'.
  120 |   useEffect(() => {
  121 |     if (!menuState) return;
> 122 |     const close = () => { setMenuState(null); setSubMenu(null); };
      |                                               ^^^^^^^^^^
```

## 根本原因

在重構 Context Menu 邏輯時，將修改/刪除功能從 Context Menu 移至活動 Popup，因此移除了 `subMenu` state（`useState<SubMenuState>(null)`）及相關的 `SubMenuState` 型別。

但 `useEffect`（監聽 click / scroll / wheel / Escape 來關閉選單）中的 `close` 函式仍保有舊的 `setSubMenu(null)` 呼叫，導致 TypeScript 找不到該 setter。

## 影響範圍

- `src/components/Calendar/index.tsx` 第 122 行

## 修正方式

將 `close` 函式中殘留的 `setSubMenu(null)` 移除：

```ts
// 修正前
const close = () => { setMenuState(null); setSubMenu(null); };

// 修正後
const close = () => { setMenuState(null); };
```

## 預防建議

重構移除 state 時，應同步全文搜尋該 setter 名稱（如 `setSubMenu`），確認所有參照均已清除，再提交。
