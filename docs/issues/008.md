# 008 — TextArea TS2769：`onChange` 型別與 `<textarea>` 不符

## 問題描述

```
TS2769: No overload matches this call.
  Type '(event: ChangeEvent<HTMLInputElement>) => void' is not assignable to
  type 'ChangeEventHandler<HTMLTextAreaElement>'.
```

`TextArea/index.tsx` 第 12 行 `onChange={onChange}` 報錯。

---

## 根本原因

Issue 007 修正 `InputProps.onChange` 為 `ChangeEvent<HTMLInputElement>` 後，`TextArea` 元件仍沿用 `InputProps`。但 `TextArea` render 的是 `<textarea>`，styled-component `StyledTextArea` 的 `onChange` 期望 `ChangeEventHandler<HTMLTextAreaElement>`，兩者衝突。

```
InputProps.onChange  →  ChangeEvent<HTMLInputElement>   ← 修正後
StyledTextArea       →  ChangeEvent<HTMLTextAreaElement> ← 不相容
```

---

## 相關檔案

| 檔案 | 說明 |
|------|------|
| `src/common/TextArea/index.tsx` | 使用 `InputProps` 但 render `<textarea>` |
| `src/common/types.ts` | `InputProps` 只定義 `HTMLInputElement` |

---

## 修復內容

`types.ts` 新增獨立的 `TextAreaProps`（`onChange` 型別為 `HTMLTextAreaElement`），`TextArea/index.tsx` 改用 `TextAreaProps`：

```ts
// types.ts（新增）
export interface TextAreaProps {
  name: string;
  placeholder: string;
  t: TFunction;
  value?: string;
  onChange: (event: React.ChangeEvent<HTMLTextAreaElement>) => void;
}
```

```tsx
// TextArea/index.tsx（修改）
import { TextAreaProps } from "../types";
const TextArea = ({ name, placeholder, onChange, t }: TextAreaProps) => ...
```
